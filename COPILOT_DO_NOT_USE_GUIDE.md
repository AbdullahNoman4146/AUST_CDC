# Copilot — What Not To Use As Guidance

This document lists sources, content types, and practices that should NOT be used as guidance when accepting code or suggestions generated by Copilot. Treat items below as strict prohibitions or high-risk signals that require human review or alternative trusted sources.

## Do Not Use

- Leaked, stolen, or unauthorized proprietary source code or internal repositories.
- Any content that contains personal data, private keys, passwords, API tokens, or other secrets.
- Unlicensed third-party code when the license is unknown or incompatible with the project.
- Code copied wholesale from paid/platinum-only documentation or vendor-only portals without an explicit license.
- Examples that require proprietary services or accounts (production credentials, commercial APIs) without secure stubs.
- Outdated or deprecated frameworks/libraries known to have critical vulnerabilities.
- Code that intentionally disables security controls (turning off authentication, removing input validation, disabling CSRF, etc.).
- Snippets that bypass privacy protections or automate data scraping in violation of terms-of-service.
- Content that contains hate speech, harassment, discrimination, or other offensive material.
- Anything that encourages illegal activity or violates applicable laws, regulations, or organizational policy.
- Heavily opinionated tutorials or blog-code that hasn't been security-audited or reviewed.
- Complex cryptographic or security-critical code (cryptography, key management) without expert review.
- Code that relies on unpublished, undocumented, or hallucinated APIs — always verify against official docs.

## Safer Alternatives / Required Actions

- Prefer official documentation, vendor SDKs, and well-maintained open-source projects with clear licensing.
- Verify licenses for any external code; if in doubt, seek legal or maintainer guidance.
- Run static analysis, security scans, and dependency checks on new code before merging.
- Replace any secrets with environment variables or secure vault references; never commit secrets.
- Require at least one human review for code suggested by Copilot when it touches security, licensing, or privacy concerns.

## Notes

- When in doubt, do not accept the suggestion verbatim. Use suggestions as a starting point and validate.
- Keep this file updated with organization-specific disallowed sources or additional rules.
